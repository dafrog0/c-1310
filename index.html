Thoughts on Web Accessibility

Web accessibility means allowing as many people as possible to use your website, including people who are older, blind, deaf, new to the internet, have motor disabilities or cognitive disabilities. This requires a level of empathy by imagining what their experience on your website would be like and allowing a smooth as possible experience for them just like any other person.

This is important because not accommodating for these people automatically reduces the number of potential users on your site. About 13% of Americans have disabilities. And it's not just about increasing traffic flow to your website. The main aspect of web accesibility is thoughtfulness which can take practice.

Thoughts on Progressive Enhancement

Progressive enhancement simply describes the approach that web developers use to ensure a basic user experience is provided to users of both advanced and older browsers. It is a type of web accesssability that has to do with the user experience and utilizes the layers of html, css, and javascript.

The antithesis for progressive enhancement would be graceful degradation, which is solely giving the best user experience to the latest, greatest browser out there with no regard for older ones. This can be harmful to your website because the main focus of this perspective does not focus on including as many browsers as possible in the user experience.
